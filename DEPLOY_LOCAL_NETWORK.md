# Развёртывание в локальной сети

Документ для передачи начальнику и системному администратору.

## Что нужно от инфраструктуры

- Один Windows-компьютер в локальной сети, который будет хостить приложение
- Python `3.11+`
- Node.js LTS
- Доступ администратора для открытия порта и настройки автозапуска

## Что передавать начальнику

- Репозиторий проекта
- Этот файл с инструкцией
- Архив релиза без `.venv`, `node_modules`, `dist`, локальных `.db` и кешей

Рекомендуемая структура передачи:
- исходники через GitHub или внутренний Git
- отдельный архив релиза для “быстрого запуска сегодня”

## Первичная установка

В PowerShell из корня проекта:

```powershell
.\deploy_local.ps1
```

Если нужен ручной сценарий, используйте команды из `README.md`.

## Ручной запуск

```powershell
.\start_backend.bat
```

Проверка:
- `http://127.0.0.1:8000/health`
- `http://127.0.0.1:8000/`

Первый логин:
- login: `admin`
- password: `admin123`

## Доступ из локальной сети

1. Узнать IP машины:

```powershell
ipconfig
```

2. Открыть порт `8000` в firewall:

```powershell
netsh advfirewall firewall add rule name="TaskTracker8000" dir=in action=allow protocol=TCP localport=8000
```

3. Передать сотрудникам адрес:

```text
http://<IP_машины>:8000
```

## Автозапуск

```powershell
.\setup_autostart.ps1
```

После этого backend будет регистрироваться в Планировщике задач как `TaskTrackerBackend`.

## Резервные копии

```powershell
.\backup_db.ps1
```

Бэкапы сохраняются в папку `backups`.

Рекомендуемый режим:
- ежедневный бэкап базы
- отдельное копирование папки проекта перед обновлением

## Обновление проекта

1. Сделать бэкап базы:

```powershell
.\backup_db.ps1
```

2. Обновить код из Git или заменить файлы проекта
3. При необходимости пересобрать frontend:

```powershell
cd frontend
npm install
npm run build
cd ..
```

4. Перезапустить backend

## Что важно знать

- База данных сейчас SQLite, поэтому это хороший вариант для небольшой команды и локальной сети, но не для высокой нагрузки
- Приложение уже умеет автоматически обновить старую структуру таблицы задач при запуске
- Для публичного интернета проект в текущем виде выкладывать не нужно; он рассчитан именно на внутреннюю сеть

## Что можно улучшить позже

- Перевести БД на PostgreSQL
- Добавить `.env`-конфигурацию
- Упаковать запуск в Docker
- Добавить reverse proxy и доменное имя внутри сети
